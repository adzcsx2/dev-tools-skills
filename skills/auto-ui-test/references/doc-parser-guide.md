# 文档解析指南

本文档定义了 `/auto-ui-test` 解析测试文档的规则和格式。

## 测试用例识别

### 识别测试用例的模式

```markdown
## 测试用例: TC-XXX
## TC-XXX
## Bug: xxx
## 测试步骤
## 复现步骤
- [ ] 待测试项
```

### 识别状态的规则

#### 跳过测试（已通过）

| 关键词 | 示例 |
|--------|------|
| PASS | `状态: PASS` |
| ✅ | `状态: ✅` |
| 已通过 | `状态: 已通过` |
| 测试通过 | `测试通过` |
| [PASS] | `[PASS] 测试用例` |
| [x] | `- [x] 测试项` |

#### 执行测试（需验证）

| 关键词 | 示例 |
|--------|------|
| FAIL | `状态: FAIL` |
| ❌ | `状态: ❌` |
| 待验证 | `状态: 待验证` |
| 修复后待验证 | `状态: 修复后待验证` |
| PENDING | `状态: PENDING` |
| [FAIL] | `[FAIL] 测试用例` |
| [ ] | `- [ ] 测试项` |
| 无状态 | `步骤: ...` (无状态字段) |

## 步骤提取

### 格式1: 编号列表

```markdown
**步骤**:
1. 打开App
2. 点击设置按钮
3. 进入关于页面
4. 验证版本号

提取为:
["打开App", "点击设置按钮", "进入关于页面", "验证版本号"]
```

### 格式2: 无序列表

```markdown
**复现步骤**:
- 打开App
- 点击Toast按钮
- 等待3秒

提取为:
["打开App", "点击Toast按钮", "等待3秒"]
```

### 格式3: Checkbox列表

```markdown
- [x] 已测试项      → 跳过
- [ ] 待测试项      → 执行
```

### 格式4: 纯文本步骤

```markdown
1. 首先点击按钮A
2. 然后等待2秒
3. 最后点击按钮B

提取为:
["首先点击按钮A", "然后等待2秒", "最后点击按钮B"]
```

## 预期结果提取

```markdown
**预期结果**: 应用不崩溃，Toast正常显示
**预期**: 显示版本号 1.0.0
**期望**: 页面跳转成功
```

## 文档示例

### 示例1: 标准测试报告

```markdown
# UI测试报告

## 测试用例: TC-001

**步骤**:
1. 点击"短Toast"按钮
2. 等待4秒
3. 点击"取消"按钮

**预期结果**: 应用不崩溃，Toast消失

**状态**: FAIL

---

## 测试用例: TC-002

**步骤**:
1. 进入设置页面
2. 点击关于
3. 检查版本号

**预期结果**: 显示版本号 2.0.0

**状态**: PASS

---

## 测试用例: TC-003

**步骤**:
1. 快速连续点击Toast按钮5次

**预期结果**: 应用不崩溃

**状态**: 待验证
```

**解析结果**:
- TC-001: FAIL → 执行
- TC-002: PASS → 跳过
- TC-003: 待验证 → 执行

### 示例2: Bug报告格式

```markdown
# Bug列表

## Bug-001: Toast连续点击崩溃

**复现步骤**:
- 打开App
- 快速连续点击Toast按钮

**预期**: 不崩溃
**实际**: 崩溃
**状态**: 待验证

## Bug-002: 设置保存失败

**复现步骤**:
1. 进入设置
2. 修改某项配置
3. 退出并重新进入

**预期**: 配置保持修改后的值
**实际**: 恢复默认值
**状态**: 已修复
```

**解析结果**:
- Bug-001: 待验证 → 执行
- Bug-002: 已修复 → 跳过（视为PASS）

### 示例3: Checklist格式

```markdown
# 测试清单

## 功能测试
- [x] 登录功能
- [x] 注销功能
- [ ] 数据同步
- [ ] 离线模式

## 性能测试
- [ ] 快速点击响应
- [ ] 大数据量加载
```

**解析结果**:
- 登录功能: [x] → 跳过
- 注销功能: [x] → 跳过
- 数据同步: [ ] → 执行
- 离线模式: [ ] → 执行
- 快速点击响应: [ ] → 执行
- 大数据量加载: [ ] → 执行

## 步骤转换规则

将自然语言步骤转换为可执行命令：

| 步骤描述 | 执行方式 |
|---------|---------|
| 点击xxx按钮 | Midscene视觉识别或ADB tap |
| 等待N秒 | sleep N |
| 快速连续点击 | ADB快速执行模式 |
| 输入xxx | ADB input text |
| 验证xxx显示 | Midscene截图验证 |
| 滑动到xxx | ADB swipe 或 Midscene |
| 进入xxx页面 | Midscene导航 |

## 错误处理

### 无法识别的格式

如果文档格式无法识别，提示用户：

```
⚠️ 无法从文档中提取测试用例，请确认文档格式正确。

支持的格式：
1. 测试报告格式（## 测试用例: TC-XXX）
2. Bug报告格式（## Bug: xxx）
3. Checklist格式（- [ ] 待测试项）

建议：参考 references/doc-parser-guide.md 了解支持的格式。
```

### 空文档或无待测用例

```
📋 文档解析完成

发现 3 个测试用例：
- 3 个已通过（跳过）
- 0 个待测试

无需执行测试，所有用例均已通过。
```
